Home Security Prim Documentation


Problems Updating?

Version 12.008+

No known problems

Version 9 to 12.007

Stand near the HS Prim and type this command before trying to update again. This will point these Prims to the new update server. You will not see a response from the Prim.

/8server cedd82b1-e1ff-b45f-b35f-d232583c0fac


Installation Problems

  • Both the PDS HS Prim and the land MUST technically have the same owner.
  • If the land is deeded you must deed the PDS HS Prim to the same group unless using Push.
  • Resetting the scripts will break them. The whole PDS HS Prim may need replacing.
  • You will need to be within 20m for the Prim to hear you and for you to see it's messages. You can also shout at it when within 100m but you might be too far away for the reply.
  • Check that 'Up' and 'Down' are not both set to zero by using /8status and look for Up and Down. Fix it with /8up=500 and /8down=500 commands.
  • When turning it on choose TurnOn from the menu and then select one of the options.

Known Second Life Problems/Bugs:

  • Push is useless beyond about 10m since v1.20 Havok4 Region updates. Don't use push for anything but tiny areas.
  • Version 9 Remote Displays no longer work due to SL changes. Use Version 8.
  • There is a Second Life "bug" that causes avatars in adjacent regions to be detected only once every 6 seconds.
  • Cross Region border information is unreliable. Attempts have been made to compensate but your mileage may vary.
  • The HS Prim will not recognise you across a Region border because it can't recognise you as an avatar rather than an object.
  • When using 'Group Safe' and 'Group Operated' Second Life will report that the HS Prim and avatars are in the same group if both have no group set.
  • Using the update server may be slow or time out. Try again later.

My Bugs:

  • Press Updates on the HS Prims main menu and see if you have the latest version.
  • I usually pay a bounty for finding my bugs if you are the first to report it.
